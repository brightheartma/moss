让 AI Agent 在 Monad 上买 Pendle PT——但每一步都留下可读的证据， 而且它永远不能替你签名。

你上次签名时，看到的是这个

风险标签在 Agent 动手之前就到了

Agent 调用 load 时，协议层直接返回这个操作的风险类别。 这不是文档里的一句承诺，是可以被程序读到的数据：

fundOut approval priceImpact

资金会流出、会产生授权、有价格冲击——三件事，建交易之前就说清楚。

失败的时候，它说人话

demo sender 0xcccc… holds 0 USDat but the swap needs 0.01.
Set MOSS_ACCOUNT to an address holding USDat, or lower MOSS_SWAP_AMOUNT.

而不是一句 execution reverted。 任何模拟警告都会中止流程——不是提示一下继续跑，是停下。

接下来

做这个 demo 的过程中发现了框架本身的一个缺口：模拟只会预充原生币， 不会给调用者 ERC-20，所以任何需要代币输入的操作都无法凭空模拟。 这个问题已作为提案提交上游。

下一步是在它之上做一个交易台，让用户不用读 calldata 就能买固定收益—— 同样的安全模型：先模拟，必确认，永不代签。
